Introduction to Sensors and Smart Sensors

Sensors detect physical inputs from the environment and convert them into electrical signals. Smart sensors also include signal conditioning, processing, and communication features.

Introduction to Actuators

Actuators receive signals from controllers and perform physical actions such as movement, heating, or sound. Examples: Motors, Buzzers, and Valves.

Robotic System

  • Sensors in Robotics: Help robots sense their environment (e.g., IR, Ultrasonic).
  • Actuators in Robotics: Convert control signals into motion (e.g., servo motors).
  • Smart Applications: Automated vacuum cleaners, robotic arms in manufacturing, and delivery robots.

Difference between Arduino, Raspberry Pi, and NodeMCU

Feature Arduino UNO Raspberry Pi NodeMCU
Type Microcontroller Single-board Computer Microcontroller with Wi-Fi
Operating System No OS (bare metal) Runs Linux-based OS No OS
Connectivity No inbuilt Wi-Fi Ethernet, Wi-Fi (via USB dongle or onboard) Built-in Wi-Fi
Programming Language C/C++ (Arduino IDE) Python, C/C++, etc. C++, Lua (Arduino IDE or ESP tools)
Use Case Sensor interfacing, robotics Full-fledged computing tasks IoT applications, Wi-Fi-based automation

Sensors

  • Motion Detection (PIR): Detects movement using infrared radiation.
  • Humidity (DHT11/DHT22): Measures moisture in the air, useful in climate monitoring.
  • Temperature (DHT11/DHT22): Captures temperature data for smart environments.
  • Light Detection (LDR): Senses light intensity, commonly used in street lights and alarms.

Wokwi Simulator

The Wokwi simulator is a web-based tool to simulate Arduino, ESP32, and other boards for testing circuits and codes without hardware.
